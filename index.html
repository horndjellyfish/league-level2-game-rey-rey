Simon Game

by: Julia Young

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I love the game Simon and I thought it would be both fun and simple to code.

2. What I Learned

I learned that the flashing should have been brighter, that the flashing should get faster as time goes on, that the score should be included in the game panel as opposed to at the very end. I also learned that in the end panel, the instructions were "press enter to restart" but pressing enter from the end panel would bring the player to the menu panel, not restarting in the game panel. I went with making the flashing brighter.

3. Architecture of My Game

I had a total of around 13 classes in my game. Nearly all of these classes were connected to my game panel class. SimonGPButton controlled all of the buttons in my game. SimonPatternManager dealt with creating and checking the patterns.

4. Technical Implementation

The game started out in a menu panel, player would press enter to switch into the game. Colors would flash, the player would click the same sequence of colors, and if the player lost, the game was over and would switch to the end panel which displayed the score. I'm proud of the computer pattern, the randomly generated pattern that the player had to follow. It was difficult to add on to a pattern and to get the timing right.

5. Most Challenging Part

The most challenging part of this project was using timers to space out the flashing of the two patterns as well as spacing out the flashing of each button.

6. Future Fixes and Updates

I had a few more suggestions from testers that I didn't implement, and I would like to code those. I would include the score in the game panel instead of at the very end and I would increase the rate of flashing as the score increased to make the game more difficult.
